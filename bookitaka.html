3. 구현 내용 요약

  • 비밀번호 찾기 시 임시 비밀번호는 Naver Mail SMTP Server를 통해 이메일로 발송됩니다.

4. 구현 내용

사용자

  • 인증과 인가는 JWT방식으로 Spring Security와 JJWT를 이용해 처리했습니다. Access Token은 발급 시 쿠키에 저장하며 HTTP Only 설정을 해 XSS 공격에 의한 Access Token 탈취 가능성을 낮추었습니다. 또한 Access Token의 만료 시간을 짧게 하고 Refresh Token을 추가해 Access Token이 탈취되었을 경우의 위험성을 낮추었습니다.

  • 사용자는 관리자, 회원, 비회원으로 구분됩니다.

  • 비밀번호 찾기 기능은 임시 비밀번호 발급 방식으로 구현했습니다. 임시 비밀번호는 난수 예측을 방지하기 위해 SecureRandom으로 생성했습니다. 생성된 임시 비밀번호는 Naver Mail SMTP Server를 통해 이메일로 발송됩니다.

결제

  • PG사 결제 모듈 연동은 portOne(I'mport)을 이용했습니다.

  • 결제가 완료된 독후활동지 및 쿠폰은 유효 기간이 있으며 유효 기간이 지나면 만료됩니다. 독후활동지 파일을 분실하였는데 만료일이 지났다면 다시 구매를 해야 다운로드가 가능합니다. 독후활동지의 유효 기간은 7일, 쿠폰의 유효 기간은 1년입니다.

장바구니

  • 장바구니 페이지에서 구매를 하면 구매할 목록을 쿠키에 저장하며 결제 페이지로 이동합니다.

쿠폰

  • 회원은 할인된 가격으로 쿠폰을 구매할 수 있으며 쿠폰 하나당 독후활동지 하나를 구매할 수 있습니다.

  • 쿠폰은 결제 페이지 내에서 사용 가능합니다.

  • 다수의 독후활동지 구매 시 쿠폰을 부분적으로 사용할 경우 쿠폰을 사용하지 않는 독후활동지의 금액만 청구됩니다.

독후활동지

  • 관리자는 독후활동지 관련 정보와 함께 이미지 파일과 PDF파일을 업로드해 독후활동지를 등록하거나 등록된 독후활동지를 수정할 수 있습니다. ISBN 서지정보 API를 이용해 원하는 책을 검색해서 쉽고 정확하게 입력할 수 있게 했습니다.

  • 독후활동지를 구매하지 않아도 첫 3 페이지를 미리보기 할 수 있습니다. PDF파일 업로드 시 미리보기 파일이 생성되고 원본 파일과 구분되어 저장됩니다. 미리보기 파일을 생성하고 저장하는 데에는 PDFBox를, 표시하는 데에는 PDFjs를 이용했습니다.

  • 독후활동지 목록 표시에 여러 정렬 기준이 필요해 QueryDSL을 이용했습니다.

  • 독후활동지 파일 다운로드 시 관리자가 아니라면 해당 독후활동지 보유 여부 확인 후 다운로드가 진행됩니다.

개별출제요청

  • 찾는 독후활동지가 없을 시 관리자에게 출제요청을 할 수 있습니다. ISBN 서지정보 API를 이용해 원하는 책을 검색해서 쉽고 정확하게 입력할 수 있게 했습니다.

  • 관리자 페이지 내에서 출제요청 확인과 답변 여부 업데이트가 가능합니다.
